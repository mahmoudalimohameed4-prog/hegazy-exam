# نظام اختبارات إلكترونية (Quiz Platform) - متطلبات كاملة للتنفيذ

## وصف المشروع
موقع ويب للاختبارات الإلكترونية (كويزات) - نظام كامل لإنشاء اختبارات مصغرة ورصد الدرجات الخاصة بكل طالب. ونظام دخول آمن باستخدام رقم الهاتف أو الرقم القومي فقط (لا بريد إلكتروني ولا أي وسيلة أخرى).

## التقنيات المطلوبة
- **Frontend:** React (مع Vite)
- **Backend:** Node.js + Express.js
- **Database:** MongoDB مع Mongoose
- **Authentication:** JWT
- **Styling:** Tailwind CSS أو أي مكتبة جاهزة

---

## المتطلبات الوظيفية

### 1. نظام الدخول الآمن
- المستخدم يسجل الدخول باستخدام **رقم الهاتف** (OTP عبر SMS) **أو** **الرقم القومي** (مع كلمة مرور).
- التسجيل المبدئي: إذا كان المستخدم جديداً، ينشئ حساباً بالمعرّف المختار تلقائياً.
- لا يُسمح باستخدام البريد الإلكتروني أو وسائل أخرى.
- المشروع في مرحلة تطوير، لذلك OTP يُطبع في كونسول الخادم.

### 2. لوحة تحكم المعلم
- إنشاء اختبار جديد (اسم، وصف، مدة زمنية بالدقائق).
- إضافة أسئلة متعددة الاختيارات (نص السؤال، 4 خيارات، تحديد الإجابة الصحيحة).
- تعديل وحذف الاختبارات.
- عرض نتائج الطلاب لكل اختبار (قائمة الطلاب ودرجاتهم).

### 3. واجهة الطالب
- عرض قائمة الاختبارات المتاحة.
- دخول الاختبار (يبدأ مؤقت فور البدء).
- الإجابة على الأسئلة (كل الأسئلة في صفحة واحدة).
- إرسال الإجابات واستلام الدرجة فوراً.
- عرض سجل الدرجات السابقة.
- الطالب لا يستطيع إعادة تقديم نفس الاختبار.

## المتطلبات غير الوظيفية
- تشفير كلمات المرور باستخدام bcryptjs
- JWT للجلسات مع وقت انتهاء
- Middleware للتحقق من التوكن والصلاحيات
- حماية API من CORS
- تصميم متجاوب مع الجوال

## هيكل قاعدة البيانات (MongoDB)

### مجموعة المستخدمين `users`
| الحقل | النوع | وصف |
|-------|-------|-----|
| identifier | String (unique) | رقم الهاتف أو الرقم القومي |
| identifierType | String | 'phone' أو 'national_id' |
| password | String | bcrypt hash (فقط إذا national_id) |
| phone | String | رقم الهاتف |
| otp | String | كود OTP مؤقت |
| otpExpires | Date | صلاحية OTP (5 دقائق) |
| role | String | 'teacher' أو 'student' |
| name | String | اسم المستخدم |
| createdAt | Date | تاريخ الإنشاء |

### مجموعة الاختبارات `quizzes`
| الحقل | النوع | وصف |
|-------|-------|-----|
| title | String | عنوان الاختبار |
| description | String | وصف الاختبار |
| timeLimit | Number | المدة بالدقائق |
| createdBy | ObjectId (ref: users) | المعلم المنشئ |
| questions | Array | مصفوفة الأسئلة |
| isActive | Boolean (default: true) | هل الاختبار نشط |
| createdAt | Date | تاريخ الإنشاء |

**هيكل السؤال الواحد:**
| الحقل | النوع | وصف |
|-------|-------|-----|
| questionText | String | نص السؤال |
| options | [String] | 4 خيارات |
| correctIndex | Number | فهرس الإجابة الصحيحة (0-3) |

### مجموعة النتائج `results`
| الحقل | النوع | وصف |
|-------|-------|-----|
| student | ObjectId (ref: users) | الطالب |
| quiz | ObjectId (ref: quizzes) | الاختبار |
| answers | [Number] | إجابات الطالب (فهارس) |
| score | Number | عدد الإجابات الصحيحة |
| total | Number | عدد الأسئلة |
| percentage | Number | النسبة المئوية |
| startedAt | Date | وقت البدء |
| finishedAt | Date | وقت الانتهاء |

## نقاط API المطلوبة (REST)

**البادئة:** `/api/v1`

### المصادقة
| الوظيفة | HTTP | المسار | الصلاحية |
|---------|------|--------|-----------|
| إرسال OTP | POST | /auth/send-otp | عام |
| التحقق من OTP | POST | /auth/verify-otp | عام |
| تسجيل بالرقم القومي | POST | /auth/register-national | عام |
| تسجيل دخول بالرقم القومي | POST | /auth/login-national | عام |
| جلب بيانات المستخدم | GET | /auth/me | مسجل الدخول |

### الاختبارات
| الوظيفة | HTTP | المسار | الصلاحية |
|---------|------|--------|-----------|
| إنشاء اختبار | POST | /quizzes | معلم فقط |
| جلب جميع الاختبارات | GET | /quizzes | مسجل الدخول |
| جلب اختبار محدد | GET | /quizzes/:id | مسجل الدخول |
| تحديث اختبار | PUT | /quizzes/:id | معلم (المالك فقط) |
| حذف اختبار | DELETE | /quizzes/:id | معلم (المالك فقط) |

### النتائج
| الوظيفة | HTTP | المسار | الصلاحية |
|---------|------|--------|-----------|
| تقديم إجابات | POST | /results/submit | طالب فقط |
| نتائج اختبار (للمعلم) | GET | /results/quiz/:quizId | معلم |
| سجل درجات الطالب | GET | /results/myresults | طالب |

## منطق الأعمال المهم

### OTP للمصادقة برقم الهاتف
1. المستخدم يدخل رقم الهاتف → يضغط إرسال OTP
2. الخادم يبحث عن مستخدم، إذا لم يوجد ينشئ واحد جديد بدور 'student'
3. يولد OTP (6 أرقام) ويحفظه مع وقت انتهاء (5 دقائق)
4. في التطوير: يطبع OTP في الكونسول
5. المستخدم يدخل OTP → الخادم يتحقق → يعيد JWT وبيانات المستخدم
6. JWT يحتوي على: userId, role, identifierType, identifier
7. صلاحية JWT: 7 أيام

### تسجيل الدخول بالرقم القومي
1. المستخدم يدخل الرقم القومي + كلمة المرور
2. إذا كان أول مرة: يسجل بكلمة مرور جديدة
3. الخادم يتحقق من التطابق → يعيد JWT

### حساب نتيجة الاختبار
1. الطالب يرسل: quizId, answers (مصفوفة فهارس), startedAt
2. الخادم يتحقق أن الطالب لم يقدم هذا الاختبار من قبل
3. يقارن كل إجابة مع correctIndex للسؤال المناظر
4. يحسب: score, total, percentage
5. يحفظ النتيجة ويعيدها للطالب

### مؤقت الاختبار
- المؤقت يدار في الواجهة الأمامية (Frontend)
- عند انتهاء الوقت، تُرسل الإجابات تلقائياً
- وقت البدء يُحفظ في قاعدة البيانات
