## ميزات إضافية مطلوبة في الواجهة
- React Router للتوجيه بين الصفحات
- React Context لحفظ بيانات المستخدم و JWT
- Axios لطلبات API مع اعتراض للتوكن
- Navigation Bar يختلف حسب الدور (معلم/طالب)
- حماية المسارات: إعادة توجيه لصفحة الدخول إذا لم يكن مصرح
- عد تنازلي واضح في صفحة تقديم الاختبار
- رسائل خطأ ونجاح واضحة للمستخدم
- تأكيد قبل حذف الاختبار
- لا يمكن للمعلم رؤية أو تعديل اختبارات معلمين آخرين

---

## متطلبات التشغيل النهائية
1. `npm install` في مجلد client ومجلد server
2. seed.js بسيط لإنشاء حساب معلم افتراضي:
   - رقم قومي: 12345678901234
   - كلمة مرور: password123
3. تشغيل الخادم: `npm run dev` على المنفذ 5000
4. تشغيل الواجهة: `npm run dev` على المنفذ 5173

## ملاحظات هامة للتنفيذ
- لا تستخدم TypeScript، استخدم JavaScript العادي (.js و .jsx)
- استخدم `"type": "module"` في package.json أو استخدم require - المهم يكون consistent
- كل الملفات تكون كاملة وجاهزة للتشغيل بدون أي تعديلات
- أضف تعليقات عربية في الأماكن المهمة
- الكود يكون نظيف ومنظم وسهل القراءة
- استخدم async/await في كل الـ controllers
- middleware التحقق: `auth` يتحقق من JWT فقط، `authorize(...roles)` يتحقق من الصلاحيات
- مسارات API:
  - `/api/v1/auth/*` للمصادقة
  - `/api/v1/quizzes/*` للاختبارات
  - `/api/v1/results/*` للنتائج
- الصلاحيات:
  - `auth` = أي مستخدم مسجل الدخول
  - `authorize('teacher')` = المعلمين فقط
  - `authorize('student')` = الطلاب فقط
